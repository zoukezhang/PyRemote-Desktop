日志显示：
1.  **连接成功** (`New connection`)
2.  **验证成功** (`Auth Success`)
3.  **连接立刻关闭** (`Connection Closed`)

这非常奇怪！通常只有**客户端主动断开**，或者**服务端崩溃**才会这样。但服务端并没有报错堆栈。

最大的嫌疑是 **截屏线程 (`stream_screen`) 出错了**。
如果在 Windows 上 `mss` 截屏库找不到显示器，或者抓取屏幕失败，它可能会导致后台任务异常，进而影响连接。

## 诊断计划
我们需要给“幕后工作者”——**截屏功能**也加上监控摄像头（日志）。

1.  **修改 `server.py`**：在 `stream_screen` 函数里增加大量 `DEBUG` 日志。
    *   检查是否找到了显示器。
    *   检查截屏是否成功。
    *   检查图片压缩是否报错。
2.  **用户操作**：您再次更新代码并运行，查看黑框里是否会不断输出 `DEBUG: Frame sent...`。
    *   如果有输出：说明截屏正常，问题在网络传输。
    *   如果没输出或报错：说明是**截屏功能**在 Windows 上水土不服。

此步骤将彻底揭开“连接即断”的真相。
